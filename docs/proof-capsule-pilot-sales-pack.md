# Proof Capsule Pilot Sales Pack

## One-liner

Proof Capsule gives a regulated team one shareable proof room where tokenised assets, off-chain documents, signed attestations, external system references, and DUAL state are bound into a verifier-ready workflow decision.

## Buyer Problem

Teams can already create tokens, documents, logs, and attestations, but the decision that relies on them is usually reconstructed manually. The gap is not another token issuer; it is a governed proof envelope that shows what was checked, what changed state, what DUAL anchored, what the source systems prove, and what still requires a live adapter or operator approval.

## Pilot Offer

Pilot room: one customer workflow, one DUAL-backed proof workspace, up to three workflow templates, public verifier pages, MCP handoff pack, operator-gated DUAL sync path, and weekly launch review.

Price guide: USD 12k setup plus USD 4k/month for the pilot period.

Best first buyers:

- Trade, treasury, insurance, compliance, tokenisation, and agent-governance teams.
- A team with one high-friction decision that already depends on multiple proof sources.
- A team willing to keep raw evidence in its own source systems while Proof Capsule stores refs, hashes, state, decisions, and DUAL anchors.

## Acceptance Gates

The pilot is accepted when:

- The selected workflow has a declared state model, required evidence types, source verifier contracts, and DUAL object/template strategy.
- Proof run score is at least 90 on the customer workflow and every hash class re-derives.
- Public verifier link integrity detects both pinned valid links and tampered links.
- DUAL write path is operator-gated, public writes are false, and readback is performed after any approved write.
- Connector readiness is explicit for every source: live, reference, or adapter needed.
- Tenant activation artifacts are generated for billing, SSO, API credentials, customer gateway, live adapters, and DUAL binding without returning secrets.
- Raw evidence remains in the customer source system or vault.

## What The Pilot Does Not Claim

- It emits customer activation artifacts, but it does not capture payment, create public-demo SSO sessions, or return API secrets through browser/MCP/API responses.
- It does not prove non-DUAL source facts live unless a tenant adapter or signed source feed is connected.
- It does not transfer tokens, move funds, execute settlement, retire credits, or perform wallet actions.
- It does not store raw evidence; it stores references, hashes, state, policy decisions, and DUAL anchors.

## Objection Handling

`We already have tokens.`
Proof Capsule does not replace the token. It wraps the token proof with policy, evidence refs, decision, state transition, and DUAL readback so another agent or reviewer can decide whether to rely on it.

`Our evidence is off-chain.`
That is the point. The capsule keeps raw evidence in the source system and binds refs, hashes, CIDs, signed IDs, and DUAL state into a governed proof room.

`Can it work outside trade?`
Yes. The model is use-case agnostic: subject, claims, evidence refs, policy gate, decision, state transition, DUAL anchor, and source verifier contracts.

`Are all adapters live now?`
No. The product is honest about this. DUAL readback is live for the canonical capsule; non-DUAL sources are labelled as reference or adapter-needed until the customer connects production adapters.

`Can an agent write to DUAL?`
Only through server-side operator-gated tools. Public verifier pages expose read/verify surfaces only.
